# Bugfix: getStagedFiles quoted-path tokenizer bypass

## Bug

`getStagedFiles` in `guya-pre-commit-review.mjs` uses `split(/\s+/)` to tokenize filenames out of combined `git add ... && git commit` commands (TOCTOU mitigation). Quoted paths like `"file with spaces.js"` split into garbage tokens (`['"file', 'with', 'spaces.js"']`), none of which match real files. Combined with `git diff --cached` returning empty (PreToolUse fires before the add runs), `isSmallChange` sees 0 staged lines → returns true → gate bypassed.

## Root Cause

Tokenizer doesn't understand shell quoting. 3-line fix, not an architecture problem.

## Fix

Replace `split(/\s+/)` with a shell-aware regex tokenizer that handles double-quoted, single-quoted, and unquoted tokens. All existing behavior preserved.

```js
const re = /"([^"]+)"|'([^']+)'|(\S+)/g;
let m;
while ((m = re.exec(addMatch[1])) !== null) {
  const token = (m[1] ?? m[2] ?? m[3]).trim();
  if (!token.startsWith('-') && token.length > 0) staged.add(token);
}
```

## Decisions

- **Keep `isSmallChange` feature**: one-line fix shouldn't require two full review passes.
- **No block on combined commands**: too restrictive, Claude always issues combined add+commit.
- **No TOCTOU mitigation redesign**: parser is correct for the common case; fix the tokenizer only.
- **Urgency LOW**: Daniel has no filenames with spaces today. Fix is prophylactic.

## Verification

4 unit tests: double-quoted path, single-quoted path, mixed quoted+unquoted, regression for existing multi-file unquoted case.
