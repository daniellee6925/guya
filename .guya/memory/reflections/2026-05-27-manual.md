**The pre-commit evidence ordering bug on the nanoclaw#3 commit.** I recorded review evidence against telos, then ran `git add && git commit` as a compound command. The Claude Code `PreToolUse` hook intercepts the **whole** Bash invocation before any of it runs — so `git add` never executed, my evidence had captured an empty tree, and the gate would have rejected the retry. I caught it from the `git status` output ("no changes added to commit" + 3 untracked) and reordered (stage → re-record evidence → commit). The rule for next time: when a `PreToolUse` hook blocks a compound command, assume *nothing* in the command ran, not just the gated subcommand. Separate staging from the gated step.
