## Part 2: What Guya Should Change

**I shipped a broken manifest format and didn't catch it before declaring the commit done.** First Constantia commit (`bd0359e`) had every row in `tasks/MANIFEST.md` missing a leading `|` — markdown table broken. The bug was in my row-prefix template; the strip regex stripped one too many pipes because I wrote `|| $id` when I needed `||| $id` to leave a leading pipe behind. The fix was a one-character change. **The reason I shipped it:** I trusted the post-commit hook to "regenerate the manifest correctly" without reading the regenerated output. I caught it only when I read the manifest after the commit landed and saw the visual breakage. **Adjustment:** when a hook regenerates a derived file and I just shipped a change to the generator, read the generated output before declaring the commit clean. Don't trust the hook on the commit that *wrote the hook*.

**I dumped options when I should have made the call.** Twice today. First time on Telos restart — "(a) wait or (b) force restart, with these caveats." You called it: "what should we do? I'm lost." Second time more subtly on the verification gate — "tomorrow's cron will fire, watch for X." That second one was OK because there was nothing to do anyway. The first wasn't. **Adjustment:** when I have enough information to make a call, make the call with reasoning and let you push back. Reserve options-dumping for cases where the tradeoff is genuinely your domain (taste, priorities, blast radius you alone know).

**I correctly diagnosed Telos's wrong self-diagnosis, but I want to be careful here.** Telos said "the overwrite protection isn't working." I read the code, traced the actual sequence (manual delete via `afd515c` cleared the slot, cron fired into empty, `fs.access` correctly threw ENOENT, tool correctly wrote), and said the bug was in `reflect-prompt.md`'s reasoning rules, not the tool guard. I was right about the diagnosis. But I was reading code Telos wrote, in a state Telos can't directly verify (Telos reads the constantia repo, not its own MCP source on disk). The takeaway isn't "I'm sharper than Telos at code reading." It's: **when an agent reports a bug, the agent's diagnosis is the bug as the agent perceives it, not necessarily the bug as it exists.** Diagnosis-from-symptoms is fallible across agents the same way it's fallible across humans. Watch for this.

**The scribe-residue working-tree pattern is annoying but I treated it as inevitable.** Every commit leaves `STATUS.md` and `.guya/evolution/last-scribe-head` dirty because the post-commit scribe writes after the commit. I documented this as "the scribe's natural baseline" and moved on. That's accurate but resigned. There's a real fix: have the scribe stage the next commit's changes during post-commit and amend, OR have the scribe write to a different file that doesn't show as dirty. Not building it tonight, but flagging it as latent friction worth scoping later.
