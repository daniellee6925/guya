## What Guya should change

**Patched the wrong layer twice — same mistake within 12 hours.** ADR-019 noon 5/14 and ADR-022 step 3 at 00:30 5/15: both INSERTed into per-session `destinations`. Both got wiped by `writeDestinations()` on next container wake. Only after the second wipe did I read `write-destinations.ts` and find the projection layer. Future-me rule: **when something I just patched gets reverted by the system, that's the system telling me I'm at the wrong layer, not a regression to schedule for later.** If I write into a DB row and the row disappears within one container lifecycle, stop. Don't re-INSERT. Read the writer code first.

**Dismissed Telos's correct self-diagnosis. Twice.** First when he said "wire up a destination" (where I was right to challenge but went too far and ignored the rest of his evidence). Second when he said *"내용이 비어있었어요"* (the content was empty) — that was a specific evidence claim, not a guess. I treated it as opinion. The lesson: **a system component diagnosing its own behavior with specific observable claims (file path, row state, content shape) is data, not opinion.** When Telos names a strip point, inspect formatTaskMessage; don't construct a more sophisticated alternative theory. I spent a debugging round on chat-sdk-bridge before coming back to formatTaskMessage where Telos had been pointing.

**`??` vs `||` matters; should be a code-review heuristic, not trivia.** ADR-021's whole fix was "change `??` to `||` at three sites." JS-specific gotcha — `??` only catches null/undefined, `||` catches all falsy. Future-me: when I see `?? null` or `?? <default>` in JS/TS code, the next question is "is the type `string | null`? could empty string flow through?"

**Drifted toward over-presentation when execution was authorized.** When Daniel says "do it now," skip the menu. Execute, then report. The AskUserQuestion for Bug #2 was the right shape (real choices, defer was a valid option). The next-step framings for "should we update ADR-019" was unnecessary — should have just done it. Calibrate: ask when there's a real decision, execute when there's just a path.

**Timezone confusion bit me once.** Read sqlite `created_at` as `2026-05-15 17:21:42` and concluded "5:21 PM" when it was 10:21 AM PT. Sqlite returns UTC by default; mini's local is PT. Caused a moment of "when did this happen?" confusion. Future-me: when reading sqlite timestamps, default-assume UTC; use `datetime(col, "localtime")` for display.

## Key decisions

- **Extract ADRs 014-022 to `docs/adrs/`** rather than tightening prose. Unbounded growth shape needs the right container, not just smaller prose.
- **Central `agent_destinations` is the durable seed layer** for any new Telos session bootstrap. Per-session `destinations` is a projection cache; never INSERT there manually.
- **Synthetic `/clear` injection** is a real operational recipe — insert chat-sdk row with `/clear` text + populated routing + `docker kill` the container. Documented in ADR-023.
- **Daemon-side fixes ship via `pnpm build` + kickstart**; container-side TS source ships via bind mount on container respawn (no docker build needed). ADR-020's keychain blocker doesn't apply to source-only changes.
- **ADR amendments** (postscript "Correction" sections) preserve original narrative + flag the correction. Better than retroactive edits or silent supersession.
