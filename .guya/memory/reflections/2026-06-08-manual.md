## Key decisions

- **Commit-only in the 10pm tick + late-reply handler in the operating contract.** A cron tick is single-turn and can't block for a reply; the durable fix puts the reply-handling in the conversational-turn contract, not the tick.
- **`isUnit123` single source of truth** over per-site `Number()` fixes — eliminates the per-site drift that *caused* guya#5, and makes the bug class regression-testable as a pure unit.
- **Fold idle-pull into the one daemon (option B), not a separate puller (option A)** — avoids a two-process `index.lock` race; one process, one heartbeat to monitor.
- **Pin the deploy key with `IdentitiesOnly=yes`** — was silently using the host `guyacode` identity via a container-path `core.sshCommand`; `IdentitiesOnly` removes the fallback so a broken key fails loudly.
- **Keep the `bb0a420` marker commit** rather than force-push it out — append-only discipline on shared truth beats tidiness.
- **Merge the T-006 conflict keeping both sides** (Telos's grade A + the laptop's completion notes) rather than picking a winner — both were valid, just concurrent.
