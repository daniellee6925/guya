- [2026-04-24] **Single PreToolUse:Bash dispatcher (ADR-012).** Claude Code 2.1.101 introduced semantic dedup of `PreToolUse` matcher entries: two entries under `matcher: "Bash"` (or distinct strings that compile to equivalent regexes) silently collapse to one. Detected after the review gate failed to block 7 unreviewed `.py` commits in `auto_eval` over a 16-day window. Fix: single `guya-pre-bash-dispatch.mjs` registered, fans out to pre-commit-review then pre-push-check as subprocesses, returns first blocking decision. Cost ~150 ms cold-start per Bash tool call. Same lesson as ADR-011 — silent rot of trusted enforcement; rely on hook execution AND verify hook execution via transcript.
- [2026-04-27] **isMain symlink fix + hook smoke test (ADR-013).** Three findings, one session: (1) `PreToolUse:Skill` auto-evidence path had silently no-op'd from the moment Claude Code's symlinked plugin install became the dispatch path — `fileURLToPath(import.meta.url)` resolves to the realpath, `process.argv[1]` keeps the symlink, equality always failed, `main()` never ran. Affected every hook with the `isMain` IIFE: pre-commit-review, correction-detect, post-commit-scribe, session-start, session-end. Fix: `realpathSync` on both sides. (2) Manual workaround (`record-review-step.mjs` + Step 0 in review SKILL.md, shipped earlier same day) removed once auto-evidence verified end-to-end. (3) New `__tests__/hooks-smoke.test.mjs` asserts `main()` runs via symlink path for every registered hook in `hooks.json`; pre-push wires it as the `guya-hook-smoke` check. Verified by reverting the realpath fix in one hook and watching the smoke fire with a clear silent-no-op error message. Same meta-pattern as ADR-011 + ADR-012: silent rot of trusted enforcement always lives in a "this can't fail" guard; the defense is a verifiable side-effect another check can read, not a smarter guard.
